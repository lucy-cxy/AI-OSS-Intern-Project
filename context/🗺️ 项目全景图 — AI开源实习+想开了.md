### r2cn.dev — AI 开源实习平台
**核心定位：** 连接 AI 开源项目（GitHub 维护者）与学生开发者，让学生通过真实 PR 贡献积累可验证的工程履历。

| 字段 | 内容 |
|------|------|
| 网址 | r2cn.dev |
| 阶段 | Demo 已上线 |
| 项目入驻 | 5 个 |
| 学生注册 | 200 个 |
| 转化率 | 20% |
| 学生激励 | 50 元/任务 |
| 启动资金 | VC 资助（希望平台帮助触达和孵化可投资项目） |
| 对外身份 | Richard Lin 主导 |

**唯一刚需：** 学生求职（CS 学生需要可验证的项目经历，AI 时代开源贡献证明的不是"会写代码"，而是"能在真实工程环境里 ship 东西"）。

**商业模式现状：** 三条独立收入线（短期养运营）→ 长期数据资产（人的成长轨迹预测模型）。付费方不是学生，候选收费方是招聘端企业。具体变现形式待数据积累到临界点（500+ 完整案例）后验证。

**r2cn 追踪的核心指标（AI 无法伪造）：**
- PR review 修改轮次（一次通过 vs 改五轮）
- Maintainer 评价（不可伪造的第三方判断）
- Issue 解决周期
- 跨模块贡献深度（只改文档 vs 碰核心模块）
- 架构讨论参与度

## 二、三个项目的协同逻辑

```
Scorecard → 项目质量过滤器
            「别浪费时间在烂项目上」
            面向学生：保证参与的项目有含金量
            面向项目方：评估报告是付费产品  
            面向 VC：周期性现金流来源

想开了   → 双端获客 + 决策指南
            面向学生：「哪个项目适合我」
            面向项目方：采访建立关系
            面向行业：赞助内容是付费产品

r2cn     → 履历生产线 + 数据资产
            面向学生：实习机会 + 50 元/任务收入
            面向企业：经过验证的候选人（变现方式待定）
            面向长期：人的成长轨迹数据
```

**内部协同机制（两个身份保持公开独立）：**
- Scorecard 评估达到 Yellow（7.0+）的项目 → 自动进入 r2cn.dev 邀请名单
- 想开了采访 = 建立项目方关系 + r2cn 获客漏斗
- r2cn 的真实贡献者数据 + Scorecard A 维度数据 → 比 GitHub Star 更完整的生态健康信号（对 VC 盈调有价值）
